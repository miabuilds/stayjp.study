# StayJP P2.5 — Web Schema Alignment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** Make App's Firestore reads/writes interoperate bidirectionally with the existing web (`stay-jp-notes` / 日本再留計劃) so that a user who learns on web and then opens App (or vice versa) sees the same SRS progress, exam date, base level, and goal level — with no migration step and no data loss.

**Why now:** Web is the source of truth (it has all the users today; App has zero). Once App ships and accumulates its own users with the wrong schema, fixing this becomes a migration headache. Aligning before launch costs ~half a day; aligning after costs days of migration code plus user data risk.

**Architecture change:** App currently writes to subdocs like `users/{uid}/srs/data` with `SyncedDoc<T> = {data, updatedAt}` wrapper. Web writes flat top-level fields on a single `users/{uid}` doc. We're switching App's persistence to match web exactly — same path, same field names (snake_case), same field shapes. The MMKV mirror keeps its own internal keys for App's convenience, but every push/pull maps to web schema.

**Non-goals:**
- No new features
- No UI changes
- Don't touch web code (web is SoT)
- Tasks 12 (Apple), 18 (rules deploy), 19, 20 stay deferred — independent of this work

**Reference web code:**
- `stay-jp-notes/index.html` lines 1378-1463 (`loadCloudData`) — merge logic to mirror
- `stay-jp-notes/index.html` lines 1483-1497 (`saveSRSCloud`, `saveAllCloud`, `SYNC_KEYS`) — write paths

## SrsEntry shape (target, web-compatible)

```ts
interface SrsEntry {
  interval: number;       // days
  ease: number;
  reviews: number;
  correct: number;
  nextReviewTs: number;   // ms epoch
  nextReview: string;     // ISO yyyy-mm-dd (kept in sync with nextReviewTs day)
  lastReviewTs?: number;  // ms epoch
  lastReview?: string;    // ISO yyyy-mm-dd (NEW — web has this; App is missing)
}
```

App's current `SrsEntry` is missing `lastReview`. Adding it is purely additive — web reads it for merge tie-breaking. P2.5 task 1 fixes this.

## Merge rules (mirror of web `loadCloudData` lines 1389-1436)

When pulling cloud:
1. **`srs_data`**: object merge. For each key:
   - If key not in local → take cloud value
   - If both exist → cloud wins if `cloud.lastReview > local.lastReview` (ISO string compare); otherwise local wins
2. **`exam_date` / `base_level` / `goal_level`**: scalar; cloud wins if cloud value is non-empty and differs (treat cloud as authoritative because they update rarely). On first cloud load, write cloud → local. Subsequent local edits push to cloud.

(Note: web's `loadCloudData` uses cloud-wins-if-newer for objects in general but its scalar handling for `exam_date`/etc is effectively "cloud overrides on first load". We mirror that. Implementation-wise: we will fetch on auth, then any later local writes push immediately via debounce.)

## Migration of existing App data (zero users today, so trivial)

Currently in App MMKV:
- `srs/data` → SyncedDoc wrapper
- `prefs/learning` → SyncedDoc wrapper containing `{baseLevel, goalLevel, examDate}`

We will:
1. Unwrap on first load (drop `.data` wrapper if present)
2. Write to new MMKV keys: `srs_data`, `exam_date`, `base_level`, `goal_level` (snake_case, matching web for grep-ability and to reduce mental cost)
3. Old keys deleted after successful migration

Since user count = 0 in App, we don't need to write Firestore-side migration. We just ensure App's first cloud read merges any existing web data correctly.

```ts
// src/lib/webSync.ts
import { doc, getDoc, setDoc } from "firebase/firestore";
import { auth, db } from "./firebase";
import type { SrsData, SrsEntry } from "../srs/types";

const DEBOUNCE_MS = 5000;

export interface WebUserDoc {
  srs_data?: SrsData;
  exam_date?: string;
  base_level?: string;
  goal_level?: string;
  // ...future fields added in later phases
}

let pendingFields: Partial<WebUserDoc> = {};
let pendingTimer: ReturnType<typeof setTimeout> | null = null;

export function schedulePushField<K extends keyof WebUserDoc>(field: K, value: WebUserDoc[K]): void {
  (pendingFields as Record<string, unknown>)[field] = value;
  if (pendingTimer) clearTimeout(pendingTimer);
  pendingTimer = setTimeout(() => { void flushAll(); }, DEBOUNCE_MS);
}

export async function flushAll(): Promise<void> {
  if (pendingTimer) { clearTimeout(pendingTimer); pendingTimer = null; }
  const payload = pendingFields;
  pendingFields = {};
  if (!Object.keys(payload).length) return;
  const uid = auth.currentUser?.uid;
  if (!uid) return;
  await setDoc(doc(db, `users/${uid}`), payload, { merge: true });
}

export async function pullWebDoc(): Promise<WebUserDoc | null> {
  const uid = auth.currentUser?.uid;
  if (!uid) return null;
  const snap = await getDoc(doc(db, `users/${uid}`));
  if (!snap.exists()) return null;
  return snap.data() as WebUserDoc;
}

/**
 * Mirror of web's loadCloudData merge for srs_data:
 * cloud wins if cloud.lastReview > local.lastReview (ISO string compare);
 * local wins otherwise. Missing keys filled from the other side.
 */
export function mergeSrsData(local: SrsData, cloud: SrsData): SrsData {
  const out: SrsData = { ...cloud };
  for (const [k, lv] of Object.entries(local)) {
    const cv = out[k];
    if (!cv) { out[k] = lv; continue; }
    const localNewer = (lv.lastReview ?? "") > (cv.lastReview ?? "");
    if (localNewer) out[k] = lv;
  }
  return out;
}
```

```ts
// src/stores/prefsStore.ts
import { getJSON, setJSON, storage } from "../lib/storage";
import { schedulePushField } from "../lib/webSync";
import type { Level } from "../types/content";
import type { BaseLevel } from "../srs/stats";

/**
 * Storage shape matches stay-jp-notes web app (flat top-level keys, snake_case).
 *   base_level: BaseLevel
 *   goal_level: Level | ""
 *   exam_date:  string (ISO yyyy-mm-dd) | ""
 */

export type PrefKey = "base_level" | "goal_level" | "exam_date";

const DEFAULTS = {
  base_level: "none" as BaseLevel,
  goal_level: "" as Level | "",
  exam_date: "",
} as const;

export function getPref<K extends PrefKey>(k: K): typeof DEFAULTS[K] {
  if (!storage.contains(k)) return DEFAULTS[k];
  // these are plain strings, stored via setJSON (so JSON.stringify "n5" → "\"n5\"")
  return getJSON<typeof DEFAULTS[K]>(k) ?? DEFAULTS[k];
}

export function setPref<K extends PrefKey>(k: K, v: typeof DEFAULTS[K]): void {
  setJSON(k, v);
  schedulePushField(k, v as never);
}
```

```ts
// src/srs/migrate.ts
import { storage } from "../lib/storage";
import type { SrsData } from "./types";

export function runLocalMigrations(): void {
  migrateSrs();
  migratePrefs();
}

function migrateSrs(): void {
  const oldKey = "srs/data";
  if (!storage.contains(oldKey)) return;
  try {
    const raw = storage.getString(oldKey) || "{}";
    const parsed = JSON.parse(raw);
    const data: SrsData = parsed?.data ?? parsed; // unwrap SyncedDoc or accept flat
    storage.set("srs_data", JSON.stringify(data));
  } catch { /* swallow */ }
  storage.delete(oldKey);
}

function migratePrefs(): void {
  const oldKey = "prefs/learning";
  if (!storage.contains(oldKey)) return;
  try {
    const raw = storage.getString(oldKey) || "{}";
    const parsed = JSON.parse(raw);
    const obj = parsed?.data ?? parsed;
    if (obj.baseLevel !== undefined) storage.set("base_level", JSON.stringify(obj.baseLevel));
    if (obj.goalLevel !== undefined) storage.set("goal_level", JSON.stringify(obj.goalLevel));
    if (obj.examDate !== undefined) storage.set("exam_date", JSON.stringify(obj.examDate));
  } catch { /* swallow */ }
  storage.delete(oldKey);
}
```
